# Hone MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the complete Hone MVP — a self-hosted fitness PWA with AI workout generation — from zero source code to a deployable application covering all 25 MVP features.

**Architecture:** Layer-by-layer construction over 9 sprints: infrastructure → auth → exercise data → profile → plan generation → workout sessions → frontend → PWA/offline → hardening. Each sprint has clear done gates and delivers working, testable software. Full spec in `ARCHITECTURE.md`.

**Tech Stack:** Bun + TypeScript + Hono (API), SvelteKit 5 Runes + Svelte (frontend, static SPA), PostgreSQL 16 + pgvector + Prisma (`@prisma/adapter-pg` mandatory), Docker Compose + Coolify (deployment on Raspberry Pi 5), vite-plugin-pwa + Workbox + Dexie.js (PWA/offline), Ollama/OpenAI-compatible (AI), Biome (lint/format).

**Reference:** `ARCHITECTURE.md` (1,710 lines) is the authoritative spec. This plan is the execution guide. Read `ARCHITECTURE.md` once before starting.

**Estimated total:** 26–35 developer-days across 9 sprints.

---

## Context & Success Criteria

**Why we're building this:** Thomas (109 kg, 190 cm, desk job, knee issues) needs a zero-friction training companion that works offline, generates personalized plans via AI, and runs on a Raspberry Pi 5 at home. Every design decision is constrained by 256 MB container RAM and a desire for no vendor lock-in.

**MVP done when:**
- User can log in, see today's workout, train, and log sets — all without an internet connection
- AI generates a 3–4 week mesocyclus in the background; rule-based fallback activates when AI is unavailable
- Safety filters (knee-safe) prevent dangerous exercises from appearing
- App installs as a PWA on mobile and desktop
- GDPR export endpoint works

```typescript
import { PrismaClient } from '@prisma/client'
import { PrismaPg } from '@prisma/adapter-pg'
import pg from 'pg'

const pool = new pg.Pool({ connectionString: process.env.DATABASE_URL })
const adapter = new PrismaPg(pool)

export const prisma = new PrismaClient({ adapter })

// Soft-delete extension: intercepts findMany/findFirst/count
export const db = prisma.$extends({
  query: {
    $allModels: {
      async findMany({ args, query }) {
        args.where = { deletedAt: null, ...args.where }
        return query(args)
      },
      async findFirst({ args, query }) {
        args.where = { deletedAt: null, ...args.where }
        return query(args)
      },
      async count({ args, query }) {
        args.where = { deletedAt: null, ...args.where }
        return query(args)
      },
    },
  },
})

// NEVER use db.model.findUnique() on soft-deletable models —
// findUnique bypasses the extension. Use findFirst({ where: { id, deletedAt: null } })
```

```typescript
import argon2 from 'argon2'
import crypto from 'crypto'
import type { PrismaClient } from '@prisma/client'

// Conscious trade-off: below OWASP (64MB/4) but Pi-safe.
// memoryCost: 32768 = 32MB, parallelism: 2, timeCost: 4
const ARGON2_OPTS = { memoryCost: 32768, parallelism: 2, timeCost: 4 }

export class AuthService {
  constructor(private db: PrismaClient) {}

  async createLocalUser(email: string, password: string) {
    if (password.length < 12) throw new Error('Password too short')
    const passwordHash = await argon2.hash(password, ARGON2_OPTS)
    return this.db.user.create({ data: { email, passwordHash } })
  }

  async verifyLocalPassword(passwordHash: string, candidate: string) {
    return argon2.verify(passwordHash, candidate)
  }

  async createSession(userId: string, isWorkoutActive = false) {
    const rawToken = crypto.randomBytes(32).toString('hex')   // 256-bit
    const sessionHash = crypto.createHash('sha256').update(rawToken).digest('hex')
    const expiresAt = new Date(Date.now() + 8 * 60 * 60 * 1000)   // 8h default

    await this.db.session.create({ data: { userId, sessionHash, expiresAt } })
    return rawToken   // returned to client, stored hashed in DB
  }

  async validateSession(rawToken: string) {
    const sessionHash = crypto.createHash('sha256').update(rawToken).digest('hex')
    const session = await this.db.session.findFirst({
      where: { sessionHash, expiresAt: { gt: new Date() } },
      include: { user: true },
    })
    return session
  }

  async invalidateSession(rawToken: string) {
    const sessionHash = crypto.createHash('sha256').update(rawToken).digest('hex')
    await this.db.session.deleteMany({ where: { sessionHash } })
  }
}
```

```typescript
import { Issuer, generators } from 'openid-client'

export class OidcService {
  private client: any

  async init() {
    const issuer = await Issuer.discover(config.OIDC_ISSUER)
    this.client = new issuer.Client({
      client_id: config.OIDC_CLIENT_ID,
      client_secret: config.OIDC_CLIENT_SECRET,
      redirect_uris: [`${config.APP_URL}/api/v1/auth/callback`],
      response_types: ['code'],
    })
  }

  generateAuthUrl(state: string, nonce: string, codeVerifier: string) {
    const codeChallenge = generators.codeChallenge(codeVerifier)
    return this.client.authorizationUrl({
      scope: 'openid email profile',
      state,
      nonce,
      code_challenge: codeChallenge,
      code_challenge_method: 'S256',
    })
  }

  async handleCallback(params: Record<string, string>, state: string, nonce: string, codeVerifier: string) {
    const tokenSet = await this.client.callback(
      `${config.APP_URL}/api/v1/auth/callback`,
      params,
      { state, nonce, code_verifier: codeVerifier },
      // Reject alg: none and symmetric algorithms
      { algorithms: ['RS256', 'ES256'] }
    )
    const claims = tokenSet.claims()
    return { sub: claims.sub, email: claims.email as string }
  }

  async handleBackchannelLogout(logoutToken: string) {
    // Verify signature using Authentik's JWKS before taking any action
    const claims = await this.client.verifyLogoutToken(logoutToken)
    const jti = claims.jti
    if (!jti) throw new Error('Missing jti in logout token')
    return { sub: claims.sub as string, jti, expiresAt: new Date(claims.exp! * 1000) }
  }
}
```

```typescript
  for (const workout of plan.workouts) {
    const first = workout.exercises[0]
    if (!first?.tags.some(t => t.category === 'CATEGORY' && t.value === 'warmup')) {
```

```typescript
      last_error = 'worker_crash',

    const [job] = await prisma.$queryRaw<AiJob[]>`

        status = 'processing',
        worker_id = ${this.workerId},
        locked_until = NOW() + INTERVAL '3 minutes',
        heartbeat_at = NOW()
      WHERE id = (
        SELECT id FROM ai_jobs
        WHERE status = 'pending'
        ORDER BY priority DESC, created_at ASC
        LIMIT 1
        FOR UPDATE SKIP LOCKED
      )
      RETURNING *
    `

        this.stopHeartbeat()
```

## Cross-Cutting Concerns

### Error Format (RFC 7807)
All API errors return Problem Details:
```json
{
  "type": "https://hone.app/errors/validation",
  "title": "Validation failed",
  "status": 422,
  "errors": [{ "field": "email", "message": "Invalid email address" }]
}
```

### Authorization Pattern
Every repository method takes `AuthenticatedPrismaContext`:
```typescript
type AuthenticatedPrismaContext = {
  userId: string
  tx?: Omit<typeof db, '$connect' | '$disconnect' | '$transaction'>
}
```
The CI grep check blocks `$queryRaw`/`$executeRaw` without `// SECURITY-REVIEW-APPROVED` comment.

### Soft-Delete Discipline
- NEVER use `db.model.findUnique()` — it bypasses the soft-delete extension
- ALWAYS use `db.model.findFirst({ where: { id, deletedAt: null } })`
- The CI grep check should catch `findUnique` on soft-deletable models

### Testing Conventions
- Integration tests hit real PostgreSQL (never mock the DB)
- Unit tests for services can use in-memory fakes for repositories
- 80% coverage target for `apps/api/src/services/`
- Run: `bun test --coverage --coverage-threshold=80`

### Bundle Discipline
- No barrel imports: `import { ChevronRight } from 'lucide-svelte'` ✓, `import * as Icons from 'lucide-svelte'` ✗
- No date libraries (use `Intl.DateTimeFormat` / `Intl.DurationFormat`)
- No chart libraries in MVP (Phase 3)
- Target budget: SvelteKit ~40 KB + Dexie ~20 KB + Lucide named ~5 KB + app ~60 KB ≈ 125 KB

### Accessibility Testing
Before each sprint sign-off with UI components:
1. Test with NVDA + Firefox (Windows) or VoiceOver + Safari (macOS/iOS)
2. Keyboard-only navigation through the sprint's new flows
3. Verify focus management on any state transitions
4. Run `axe-core` in Playwright tests for automated WCAG checks
