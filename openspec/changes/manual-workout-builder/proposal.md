## Why

Today a user can only train from a workout template generated by the rule
engine or AI; there is no way to assemble a one-off workout from exercises
of their own choosing. This change adds that capability as a standalone,
not-saved session тАФ the smallest, cheapest slice of "manual workout editing"
тАФ without yet building persistent template editing or custom exercise
creation, both of which need separate, larger changes.

## What Changes

- Add a web UI to pick exercises (from the existing global exercise
  catalog) and assemble them, in order, with per-exercise `sets`, `reps` or
  `durationSecs`, and `restSecsOverride`, into a single ad-hoc workout
  session.
- Extend `ExerciseLog` with its own optional `sets`, `reps`,
  `durationSecs`, `restSecsOverride`, and `groupId` columns, used only when
  there is no matching `WorkoutTemplateExercise` row (i.e. for ad-hoc
  entries). Templated sessions are unaffected тАФ they keep resolving these
  values from the template exactly as today.
- Make `WorkoutSessionService.createSession`'s `templateId` accept `null`
  for ad-hoc sessions (the underlying `WorkoutSession.templateId` and
  `mesocyclusId` columns are already nullable in the schema; only the
  service/route layer currently forces a real template).
- Update `/workout/today`'s active-session lookup
  (`findActiveSession` in `apps/api/src/routes/workout.routes.ts`), which
  today returns `null` whenever a session has no `template`, so a
  templateless ad-hoc session can be displayed and resumed like any other.
- Extend the offline queue contract (`PendingSessionCreateOp` in
  `apps/web/src/lib/db/offline-store.ts`, replayed by `flushOp` in
  `apps/web/src/lib/sync.ts`) to carry the new per-exercise fields and a
  nullable `templateId`, so ad-hoc session creation queues and replays
  offline exactly like templated session creation does today.
- Out of scope for this change, explicitly: persistent `WorkoutTemplate`
  editing (add/remove/reorder exercises on a reusable template), custom
  exercise creation (`Exercise.ownerId`/`isGlobal`), image upload, and
  content moderation. These are each their own future, sequenced change.
  Alternating-group authoring (`groupId`) is wired into the new fields so
  it can be used once `alternating-exercise-sets` ships, but this change
  does not depend on that change being implemented first.

## Capabilities

### New Capabilities
- `ad-hoc-workout-sessions`: Defines how a user assembles a one-off,
  non-template-backed workout session from existing exercises, how its
  per-exercise configuration is stored and resolved, and how it behaves
  through the existing offline queue and active-session read path.

### Modified Capabilities
(none тАФ `offline-workout-resilience`'s existing requirements already cover
session creation as an offline-queueable write; this change extends that
operation's payload shape without changing any existing requirement's
normative text)

## Impact

- **Schema**: `apps/api/prisma/schema.prisma` тАФ additive nullable `sets`,
  `reps`, `durationSecs`, `restSecsOverride`, `groupId` columns on
  `ExerciseLog`; new migration. No change to `WorkoutTemplate`,
  `WorkoutTemplateExercise`, or the `WorkoutLabel` enum.
- **API**: `apps/api/src/services/workout-session.service.ts` and
  `apps/api/src/repositories/workout-session.repo.ts` (`createSession`
  accepts nullable `templateId` and per-exercise config),
  `apps/api/src/routes/workout-session.routes.ts` (request schema),
  `apps/api/src/routes/workout.routes.ts` (`findActiveSession` template-
  optional lookup and resolution).
- **Web**: new exercise-picker UI and route (reusing form/validation
  patterns from `apps/web/src/routes/onboarding/+page.svelte` and the
  `request<T>()` client in `apps/web/src/lib/api.ts`), reusing
  `apps/api/src/routes/equipment.routes.ts`'s CRUD shape as a structural
  reference where applicable.
- **Offline sync**: `apps/web/src/lib/db/offline-store.ts`
  (`PendingSessionCreateOp` payload), `apps/web/src/lib/sync.ts`
  (`flushOp` replay) тАФ both must carry the new fields and nullable
  `templateId`.
- **Unaffected**: `apps/api/src/services/rule-engine.service.ts`, AI plan
  generation, any future plan/template editor or custom-exercise UI.
